# 一、YAML 数据类型（Data Types）

YAML 数据类型分为：

- **标量类型（Scalars）**：单一数据（单值）
- **复合类型（Collections）**：包含多个数据（结构体）

## 1.1 标量类型（Scalar）

标量是不可再分的数据单元。YAML 中的标量类型包括：

| 类型       | 示例                             | 特点                                                         |
|-----------|--------------------------------|------------------------------------------------------------|
| 字符串     | `"hello"`、`'world'`、`plain text` | 可裸写、单引号、双引号、块风格写，支持多种形式 |
| 整数       | `123`、`-456`                     | 正负整数，支持多进制（8进制、16进制）                       |
| 浮点数     | `3.14`、`-0.01`、`.inf`            | 小数，特殊值（如无穷 `.inf`、非数 `.nan`）                  |
| 布尔值     | `true`、`false`、`yes`、`no`       | 多种语义表示，同义词识别                                     |
| 空值       | `null`、`~`、留空                 | 表示无数据                                                   |
| 时间/日期  | `2024-04-27`、`2024-04-27T12:00:00Z` | 自动推断为日期时间对象                                       |

---

## 1.2 复合类型（Collection）

复合类型可以嵌套标量或其它复合类型，主要包括：

- **映射（Mapping）**：键值对集合（类似 JSON 的对象）
- **序列（Sequence）**：有序列表集合（类似 JSON 的数组）

# 二、标量（Scalar）细节全览

## 2.1 字符串（String）

### 2.1.1 裸字符串（Plain Style）

```yaml
key: hello world
```
- 支持直接写，不需要引号。
- **注意：** 遇到冒号 `:`、特殊符号（如 `#`）时，需要引号。

错误示例：
```yaml
key: this:will:break  # 错误，冒号未引号包围
```

### 2.1.2 双引号字符串（Double-Quoted Style）

```yaml
key: "hello \n world"
```
- 支持转义字符：
  - `\n`（换行）
  - `\t`（制表符）
  - `\\`（反斜线）
  - `\"`（双引号）
- 允许内嵌特殊字符。

### 2.1.3 单引号字符串（Single-Quoted Style）

```yaml
key: 'hello ''world'''
```
- 不支持转义。
- `''`（两个连续单引号）表示一个单引号字符 `'`。

---

## 2.2 多行字符串（块风格 Block Scalars）

### 2.2.1 保留换行 `|`（Literal Style）

```yaml
content: |
  line1
  line2
```
解析为：
```
line1\nline2\n
```

### 2.2.2 折叠换行 `>`（Folded Style）

```yaml
content: >
  line1
  line2
```
解析为：
```
line1 line2\n
```

#### 块修饰符（Block Chomping Indicator）
在 `|` 或 `>` 后面可加符号：

- `|+`：保留最后的换行
- `|-`：去除最后的换行
- `|`：默认保留一个换行

示例：
```yaml
text: |+
  hello
  world
```
表示：
```
hello\nworld\n\n
```

---

## 2.3 布尔值（Boolean）

| 文本值 | 解析为 |
|--------|--------|
| `true`, `True`, `TRUE`, `yes`, `Yes`, `YES`, `on`, `On`, `ON` | 真 (`true`) |
| `false`, `False`, `FALSE`, `no`, `No`, `NO`, `off`, `Off`, `OFF` | 假 (`false`) |

注意：布尔值大小写不敏感。

---

## 2.4 空值（Null）

空值有三种写法：

```yaml
key1: null
key2: ~
key3: 
```
都会被解析成 `null`。

---

## 2.5 数字（Number）

### 整数（Integer）
```yaml
decimal: 123
negative: -45
octal: 0o17
hex: 0x1F
```

### 浮点数（Float）
```yaml
float: 3.1415
negative_float: -2.718
exp_float: 1e6
inf_float: .inf
nan_float: .nan
```

---

## 2.6 时间日期（Timestamp）

```yaml
date: 2025-04-27
datetime: 2025-04-27T14:00:00Z
```
自动识别为时间类型（而不是普通字符串）。

# 三、复合数据类型（Collection）

---

## 3.1 映射（Mapping）

**映射（Map）**是**键值对**结构。

### 3.1.1 块风格（Block Style）

```yaml
person:
  name: Alice
  age: 30
```

### 3.1.2 流风格（Flow Style）

```yaml
person: { name: Alice, age: 30 }
```

### 3.1.3 树状结构图

```
person
├── name: Alice
└── age: 30
```

---

## 3.2 序列（Sequence）

**序列（Sequence）**是**有序数据列表**。

### 3.2.1 块风格（Block Style）

```yaml
colors:
  - red
  - green
  - blue
```

### 3.2.2 流风格（Flow Style）

```yaml
colors: [red, green, blue]
```

### 3.2.3 树状结构图

```
colors
├── red
├── green
└── blue
```

---

## 3.3 复合嵌套示例（高阶组合）

```yaml
person:
  name: Alice
  age: 30
  contact:
    email: alice@example.com
    phones:
      - 12345678
      - 87654321
  hobbies:
    - reading
    - painting
    - music
```

对应树状图：

```
person
├── name: Alice
├── age: 30
├── contact
│   ├── email: alice@example.com
│   └── phones
│       ├── 12345678
│       └── 87654321
└── hobbies
    ├── reading
    ├── painting
    └── music
```

# 四、YAML 语法规则（Syntax Rules）

## 4.1 缩进（Indentation）

- **必须使用空格**（不能使用 Tab）
- 通常建议使用 2 个空格，也可用 4 个空格
- 同一层次缩进必须统一

---

## 4.2 注释（Comment）

```yaml
# 这是注释
key: value  # 也是注释
```
- 注释以 `#` 开始，直到行尾。

---

## 4.3 文档分隔（Multi-Document）

- `---`：开始新文档
- `...`（可选）：结束文档

示例：
```yaml
---
key: value1
---
key: value2
...
```

---

## 4.4 锚点与引用（Anchors and Aliases）

### 4.4.1 锚点定义 `&`

```yaml
defaults: &default
  adapter: postgres
  host: localhost
```

### 4.4.2 引用使用 `*`

```yaml
development:
  <<: *default
  database: dev_db
```

`<<: *default`：合并 `default` 的内容到 `development`。

---

## 4.5 特殊符号用法

- **?**：复杂键（用作 key 的是复合结构）
- **:**：键值分隔符，后面必须跟空格
- **-**：序列项目，后跟空格

---

## 4.6 合并键（Merge Keys）

使用 `<<` 关键字将一个映射插入到另一个映射中。

示例：
```yaml
defaults: &defaults
  name: default
  role: user

admin:
  <<: *defaults
  role: admin
```

结果：
```yaml
admin:
  name: default
  role: admin
```
