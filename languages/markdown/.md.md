# Markdown  Syntax

## 一、基本语法

### 1. 标题 (Headings)
#### 实现方式
- **方式 1：使用 `#` 符号**
  - 语法：`# 标题内容`
  - `#` 的数量表示标题级别（1-6级）。
  - 示例：
    ```
    # 一级标题
    ## 二级标题
    ### 三级标题
    #### 四级标题
    ##### 五级标题
    ###### 六级标题
    ```
  - 输出：
    # 一级标题
    ## 二级标题
    ### 三级标题
    #### 四级标题
    ##### 五级标题
    ###### 六级标题
  - 细节：
    - `#` 后必须跟一个空格，否则可能无效。
    - 支持 1-6 级，超过 6 个 `#` 通常被视为普通文本。
    - 可在标题后加任意数量的 `#`（不影响级别），如 `## 标题 ##`。

- **方式 2：使用下划线（仅限一级和二级标题）**
  - 语法：在标题文本下一行使用 `=`（一级）或 `-`（二级）。
  - 示例：
    ```
    一级标题
    =======

    二级标题
    -------
    ```
  - 输出：

    一级标题
    ===
    一级标题
    =======

    二级标题
    ---
    二级标题
    -------

  - 细节：
    - 下划线长度无需与标题文本长度一致，至少 3 个字符即可。
    - 标题文本与下划线之间不能有空行。
    - 仅适用于一级和二级标题，其他级别无此方法。

#### 注意事项
- 两种方式可以混用，但建议保持一致性。
- 某些处理器（如 GFM）可能对 `#` 后的额外字符（如空格或 `#`）有不同处理。

---

### 2. 段落 (Paragraphs)
#### 实现方式
- **方式 1：直接输入文本**
  - 语法：连续文本视为一个段落，段落间用空行分隔。
  - 示例：
    ```
    这是一个段落。

    这是另一个段落。
    ```
  - 输出：
    这是一个段落。

    这是另一个段落。
  - 细节：
    - 无空行分隔的换行会被视为同一段落，渲染时可能连接为一行。

#### 注意事项
- 段落首行通常不缩进，若需缩进可使用代码块或手动空格（但空格可能被忽略）。

---

### 3. 换行 (Line Breaks)
#### 实现方式
- **方式 1：在行末加两个或更多空格**
  - 语法：行末加至少 2 个空格，然后 Enter。
  - 示例：
    ```
    第一行
    第二行
    ```
  - 输出：
    第一行
    第二行
  - 细节：
    - 空格数量必须 ≥ 2，1 个空格无效。
    - 部分处理器可能忽略此规则，需测试。

- **方式 2：使用 HTML `<br>` 标签**
  - 语法：直接插入 `<br>`。
  - 示例：
    ```
    第一行<br>
    第二行
    ```
  - 输出：
    第一行<br>
    第二行
  - 细节：
    - Markdown 支持内联 HTML，`<br>` 是标准换行标签。
    - 不推荐频繁使用，破坏 Markdown 纯文本特性。

#### 注意事项
- 单次 Enter（无空格）通常不产生换行，仅分段。
- 换行效果可能因渲染器不同而异。

---

### 4. 强调 (Emphasis)
#### 实现方式
- **斜体 (Italic)**
  - **方式 1：单 `*`**
    - 语法：`*文本*`
    - 示例：`*斜体*` → *斜体*
  - **方式 2：单 `_`**
    - 语法：`_文本_`
    - 示例：`_斜体_` → _斜体_
  - 细节：
    - 符号两侧不能有空格，如 `* 斜体 *` 无效。
    - `*` 和 `_` 效果相同，但 `*` 更常用。

- **粗体 (Bold)**
  - **方式 1：双 `*`**
    - 语法：`**文本**`
    - 示例：`**粗体**` → **粗体**
  - **方式 2：双 `_`**
    - 语法：`__文本__`
    - 示例：`__粗体__` → __粗体__
  - 细节：
    - 同上，符号两侧无空格。

- **粗斜体 (Bold Italic)**
  - **方式 1：三 `*`**
    - 语法：`***文本***`
    - 示例：`***粗斜体***` → ***粗斜体***
  - **方式 2：三 `_`**
    - 语法：`___文本___`
    - 示例：`___粗斜体___` → ___粗斜体___
  - **方式 3：混合使用**
    - 语法：`**_文本_**` 或 `*__文本__*`
    - 示例：`**_粗斜体_**` → **_粗斜体_**
  - 细节：
    - 混合方式需注意符号顺序，内外层必须匹配。

#### 注意事项
- 嵌套强调（如 `*外层 _内层_*`）在部分处理器中支持，效果为外层斜体、内层粗斜体。
- 避免在强调符号与文本间加空格。

### 5. 引用 (Blockquotes)
#### 实现方式
- **方式 1：使用 `>`**
  - 语法：在每行前加 `>`。
  - 示例：
    ```
    > 这是一个引用。
    > 第二行。
    ```
  - 输出：
    > 这是一个引用。
    > 第二行。
  - 细节：
    - 多行引用可只在首行加 `>`，但建议每行都加以确保兼容性。

- **嵌套引用**
  - 语法：使用多个 `>`。
  - 示例：
    ```
    > 外层引用
    >> 内层引用
    >>> 更深一层
    ```
  - 输出：
    > 外层引用
    >> 内层引用
    >>> 更深一层

- **引用中包含其他元素**
  - 示例：
    ```
    > ## 引用中的标题
    > - 列表项
    >   1. 子列表项
    ```
  - 输出：
    > ## 引用中的标题
    > - 列表项
    >   1. 子列表项

#### 注意事项
- 引用内支持大多数 Markdown 语法。
- 空行会中断引用，需重新加 `>`。

---

### 6. 列表 (Lists)
#### 无序列表 (Unordered Lists)
- **实现方式**
  - **方式 1：使用 `-`**
    - 示例：`- 项目`
  - **方式 2：使用 `*`**
    - 示例：`* 项目`
  - **方式 3：使用 `+`**
    - 示例：`+ 项目`
  - 示例：
    ```
    - 项目 1
    - 项目 2
      - 子项目 2.1
    * 项目 3
    + 项目 4
    ```
  - 输出：
    - 项目 1
    - 项目 2
      - 子项目 2.1
    * 项目 3
    + 项目 4
  - 细节：
    - 符号后需加空格。
    - 嵌套需缩进 2-4 个空格（推荐 2 个，部分处理器要求 4 个）。

#### 有序列表 (Ordered Lists)
- **实现方式**
  - **方式 1：数字加 `.`**
    - 语法：`数字. 文本`
    - 示例：
      ```
      1. 第一项
      2. 第二项
         1. 子项 2.1
      3. 第三项
      ```
    - 输出：
      1. 第一项
      2. 第二项
         1. 子项 2.1
      3. 第三项
  - 细节：
    - 数字不必连续，渲染时自动排序（如 `1. 5. 3.` 仍显示为 `1. 2. 3.`）。
    - 嵌套同上，缩进 2-4 个空格。

- **混合列表**
  - 示例：
    ```
    1. 有序项
       - 无序子项
    - 无序项
      1. 有序子项
    ```
  - 输出：
    1. 有序项
       - 无序子项
    - 无序项
      1. 有序子项

#### 注意事项
- 列表项内可包含段落、代码块等，需额外缩进。
- 列表后接其他内容需加空行，否则可能解析错误。

### 7. 代码 (Code)
#### 行内代码 (Inline Code)
- **实现方式**
  - 语法：用单个反引号 `` ` `` 包裹。
  - 示例：`` `print("Hello")` `` → `print("Hello")`
  - 细节：
    - 若代码中包含反引号，用双反引号包裹并在两侧加空格，如 ``` ``code` `` ``` → ``code` ``。

#### 代码块 (Code Blocks)
- **方式 1：三反引号**
  - 语法：
    ```
    ```
    代码内容
    ```
    ```
  - 示例：
    ```
    ```
    def hello():
        print("Hello")
    ```
    ```
  - 输出：
    ```
    def hello():
        print("Hello")
    ```
  - 指定语言：
    - 在首行反引号后加语言名，如 ```python。
    - 示例：
      ```
      ```python
      print("Hello")
      ```
      ```

- **方式 2：缩进 4 个空格**
  - 语法：每行缩进至少 4 个空格。
  - 示例：
    ```
        def hello():
            print("Hello")
    ```
  - 输出：
    ```
    def hello():
        print("Hello")
    ```
  - 细节：
    - 嵌套需额外缩进（如列表中的代码块需 6-8 个空格）。

#### 注意事项
- 三反引号方式更现代，缩进方式较传统。
- 语言高亮依赖渲染器支持。

---

### 8. 水平线 (Horizontal Rules)
#### 实现方式
- **方式 1：三个或更多 `-`**
  - 示例：`---` 或 `----`
- **方式 2：三个或更多 `*`**
  - 示例：`***` 或 `****`
- **方式 3：三个或更多 `_`**
  - 示例：`___` 或 `____`
- 输出：
  ---
  ***
  ___
- 细节：
  - 符号间可加空格，如 `* * *`，效果相同。
  - 必须单独一行，前后加空行以避免与标题混淆。

---

### 9. 链接 (Links)
#### 实现方式
- **方式 1：内联链接**
  - 语法：`[文本](URL "可选标题")`
  - 示例：
    ```
    [Google](https://www.google.com "访问 Google")
    ```
  - 输出：[Google](https://www.google.com "访问 Google")

- **方式 2：引用式链接**
  - 语法：
    ```
    [文本][标识]
    [标识]: URL "可选标题"
    ```
  - 示例：
    ```
    [Google][1]
    [1]: https://www.google.com "访问 Google"
    ```
  - 输出：[Google][1]
    [1]: https://www.google.com "访问 Google"

- **方式 3：自动链接**
  - 语法：直接写 URL。
  - 示例：`https://www.google.com` → https://www.google.com

- **方式 4：角括号包裹**
  - 语法：`<URL>` 或 `<邮箱>`
  - 示例：`<https://www.google.com>` → <https://www.google.com>

#### 注意事项
- 标题部分可选，用引号包裹。
- URL 可为相对路径（如 `./file.md`）。

---

### 10. 图片 (Images)
#### 实现方式
- **方式 1：内联图片**
  - 语法：`![替代文本](图片URL "可选标题")`
  - 示例：
    ```
    ![示例图片](https://example.com/image.jpg "图片标题")
    ```
  - 输出：![示例图片](https://example.com/image.jpg "图片标题")

- **方式 2：引用式图片**
  - 语法：
    ```
    ![替代文本][标识]
    [标识]: 图片URL "可选标题"
    ```
  - 示例：
    ```
    ![示例图片][img1]
    [img1]: https://example.com/image.jpg "图片标题"
    ```

#### 注意事项
- 替代文本用于图片无法加载时显示。
- 不支持直接调整图片大小，需用 HTML `<img>` 标签。

---

### 11. 转义字符 (Escaping Characters)
#### 实现方式
- 语法：用反斜杠 `\` 转义。
- 示例：`\*文字\*` → \*文字\*
- 可转义字符：
  ```
  \   `   *   _   {   }   [   ]   (   )   #   +   -   .   !   |   :
  ```
- 细节：
  - HTML 实体（如 `&amp;`）也可用于转义。

---

## 二、扩展语法
这些功能在某些 Markdown 处理器（如 GFM、Pandoc）中支持。

### 1. 表格 (Tables)
#### 实现方式
- **基本表格**
  - 语法：
    ```
    | 表头1 | 表头2 |
    |-------|-------|
    | 单元1 | 单元2 |
    ```
  - 输出：
    | 表头1 | 表头2 |
    |-------|-------|
    | 单元1 | 单元2 |

- **对齐方式**
  - 左对齐：`:---`
  - 居中：`:---:`
  - 右对齐：`---:`
  - 示例：
    ```
    | 左对齐 | 居中  | 右对齐 |
    |:-------|:-----:|-------:|
    | a      | b     | c      |
    ```
  - 输出：
    | 左对齐 | 居中  | 右对齐 |
    |:-------|:-----:|-------:|
    | a      | b     | c      |

- **细节**
  - 管道符 `|` 可在首尾省略，如 `表头1 | 表头2`。
  - 分隔线中 `-` 数量不限，至少 1 个。

---

### 2. 删除线 (Strikethrough)
#### 实现方式
- 语法：`~~文本~~`
- 示例：`~~删除文字~~` → ~~删除文字~~

---

### 3. 任务列表 (Task Lists)
#### 实现方式
- 语法：`- [ ]`（未完成）或 `- [x]`（已完成）
- 示例：
  ```
  - [ ] 任务 1
  - [x] 任务 2
  ```
- 输出：
  - [ ] 任务 1
  - [x] 任务 2
- 细节：
  - `[ ]` 和 `[x]` 间必须有空格。

---

### 4. 脚注 (Footnotes)
#### 实现方式
- 语法：
  ```
  文本[^标识]
  [^标识]: 脚注内容
  ```
- 示例：
  ```
  这里有脚注[^1]。
  [^1]: 这是脚注内容。
  ```
- 输出：
  这里有脚注[^1]。
  [^1]: 这是脚注内容。
- 细节：
  - 标识可为数字或文字，脚注定义可放在文档任意位置。

---

### 5. 高亮 (Highlight)
#### 实现方式
- 语法：`==文本==`
- 示例：`==高亮文字==` → ==高亮文字==
- 细节：仅部分处理器（如 Pandoc）支持。

---

### 6. 表情符号 (Emoji)
#### 实现方式
- **方式 1：短代码**
  - 语法：`:短代码:`
  - 示例：`:smile:` → 😊
- **方式 2：直接 Unicode**
  - 示例：直接输入 😊

---

### 7. 定义列表 (Definition Lists)
#### 实现方式
- 语法（Pandoc 风格）：
  ```
  术语
  : 定义
  ```
- 示例：
  ```
  Markdown
  : 一种轻量级标记语言
  ```
- 输出：
  Markdown
  : 一种轻量级标记语言

---

### 8. 上标和下标 (Superscript and Subscript)
#### 实现方式
- **上标**
  - 语法：`^文本^`
  - 示例：`x^2^` → x^2^
- **下标**
  - 语法：`~文本~`
  - 示例：`H~2~O` → H~2~O
- 细节：需特定处理器（如 Pandoc）支持。

---

## 三、其他细节与注意事项
1. **HTML 支持**
   - Markdown 允许嵌入 HTML 标签，如 `<div>`、`<img>`。
   - 示例：`<img src="url" width="100">`
2. **空白行**
   - 空行用于分隔不同元素（如段落与列表），无空行可能导致解析错误。
3. **缩进敏感**
   - 代码块、嵌套列表等需严格缩进。
4. **兼容性**
   - 扩展语法依赖具体处理器，未支持的功能可能显示为纯文本。
